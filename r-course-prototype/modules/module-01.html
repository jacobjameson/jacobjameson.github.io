First commands

R becomes less intimidating once learners see code as small, inspectable steps.

This module keeps the original motivation for learning R, then moves quickly into assignment, arithmetic, vectors, Boolean logic, and object types.

  • Explain why R is useful for data analysis and public policy work.
  • Create named objects with `<-` and inspect their values.
  • Use vectors, comparisons, and `mode()` to reason about basic data types.

Concepts

The first mental model: create, inspect, compare.

Create

Assignment stores a value under a name: `income <- 52000`.

Inspect

Typing an object name asks R to print its current value.

Compare

Comparisons return `TRUE` or `FALSE`, which become the foundation for filtering data.

Try in Browser

Practice the first coding moves.

Use `Run` to experiment. Use `Check` when you think your code satisfies the prompt. Completed checks are saved in this browser.

Assignment and arithmetic

Create `a`, calculate `b`, and make sure `b` is exactly 10.

`3^2` is 9, and adding 1 gives 10. The object must be named `b`.

Run or check your code.

Vectors

Create a four-number vector named `scores` whose average is greater than 85.

Change at least one number so the average is above 85, then print `mean(scores)`.

Run or check your code.

Characters and paste0

Use two character objects and `paste0()` to print `Hello, world!`.

`paste0()` does not add spaces. Include the comma, space, and exclamation point in one of the strings.

Run or check your code.
Do in RStudio

Move the same moves into a script.

  1. Create a script named `module-01-first-commands.R` in your course folder.
  2. Add one comment at the top explaining what the script practices.
  3. Create at least three objects: one number, one character string, and one logical value.
  4. Use `mode()` on each object and run the script from top to bottom.
Mini-Project

Policy analyst warmup

Scenario: A policy team is tracking a small set of program enrollment counts before working with a full dataset.

Task: Create a vector of four enrollment counts, calculate the average, and write one comparison that checks whether the average is above a target.

Interpretation: In one sentence, explain what your `TRUE` or `FALSE` result means.

Future Rmd submission: A short Rmd with code, printed output, and a one-sentence interpretation.
